# Discovery

`organvm/metasystem-master` should be promoted into the ranked value tier because its highest latent value is a reusable live co-creation engine: the repo already contains a real Express/Socket.io core with `/health`, `/session`, and `/values` endpoints, separated `/audience` and `/performer` namespaces, tested weighted consensus and parameter-bus modules, performer override semantics, Docker/infra assets, a static product surface, and a working generative-music proof path that can turn audience phones into a collective control surface while preserving performer authority. The honest boundary is that several advertised surfaces are not yet product-depth (`packages/client-sdk` is README-only, `packages/audio-synthesis-bridge` source files are empty, and most non-music examples are scaffolded), so the ranked value is not a finished SDK suite; it is the core audience-consensus backplane and demonstrable participatory-performance product wedge for ORGAN-II and downstream commerce. Single best first task: ship a reproducible alpha demo that starts the core engine plus the generative-music audience/performer flow from a fresh install, passes the core-engine consensus/bus tests in CI, and documents the exact local-to-venue runbook as the first publishable package boundary.
