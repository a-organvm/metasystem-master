# Discovery

`organvm/metasystem-master` should be promoted into the ranked value tier because its highest latent value is a reusable audience-consensus backplane for live co-creation: the repo already contains a real Express/Socket.io core engine with `/health`, `/session`, and `/values` endpoints, separate `/audience` and `/performer` namespaces, typed parameter-bus and weighted-consensus modules with Vitest coverage, performer override semantics, React audience/performer UI components, Docker/GCP/nginx deployment scaffolding, and a working generative-music server/public proof that can turn phones into a collective control surface for concerts, classrooms, installations, and grant-funded venue pilots. The honest boundary is that it is not yet a finished product suite: `packages/client-sdk` is README-only, `packages/audio-synthesis-bridge` source files are empty, and several examples/tests are scaffold depth; the value is the tested core backplane plus the clearest product wedge. Single best first task: ship a reproducible alpha demo that starts the core engine and the generative-music audience/performer flow from a fresh install, adds non-empty smoke/integration tests for that demo, and documents the exact local-to-venue runbook as the first publishable package boundary.
